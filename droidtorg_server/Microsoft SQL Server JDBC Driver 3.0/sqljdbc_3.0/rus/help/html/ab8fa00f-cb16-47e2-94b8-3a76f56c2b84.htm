Общие сведения о различиях типов данных

Есть целый ряд различий между типами данных языка программирования Java и типами данных SQL Server. Драйвер JDBC Microsoft SQL Server помогает уменьшить значимость этих различий с помощью различных преобразований.

Символьные типы

Типами данных символьных строк JDBC являются CHAR, VARCHAR и LONGVARCHAR. Начиная с версии 2.0 драйвера JDBC Microsoft SQL Server в драйвере JDBC предусмотрена поддержка для API-интерфейса JDBC 4.0. В версии JDBC 4.0 типами данных символьных строк JDBC могут также быть NCHAR, NVARCHAR и LONGNVARCHAR. Эти новые типы символьных строк поддерживают собственные символьные типы Java в формате Юникода и исключают необходимость выполнять какие-либо преобразования из ANSI в Юникод или из Юникода в ANSI.

Тип Описание

Фиксированная длина

Типы данных SQL Server char и nchar непосредственно соответствуют типам CHAR и NCHAR JDBC. Они представляют собой типы фиксированной длины с заполнением, предоставляемым сервером, при условии, что для столбца установлено значение SET ANSI_PADDING ON. Заполнение всегда включено для nchar, а что касается char, то в случае, если на сервере заполнение для столбцов символьных типов не предусмотрено, то заполнение добавляет драйвер JDBC.

Переменная длина

Типы SQL Server varchar и nvarchar непосредственно соответствуют типам VARCHAR и NVARCHAR JDBC.

Long

Типы SQL Server text и ntext соответствуют типам LONGVARCHAR и LONGNVARCHAR JDBC. Эти типы являются устаревшими, начиная с версии SQL Server 2005, и вместо них нужно использовать типы больших значений varchar(max) и nvarchar(max).

Использование методов update<Numeric Type> и updateObject (int, java.lang.Object) по отношению к серверным столбцам text и ntext завершается с ошибками. Однако использование метода setObject с указанным символьным типом преобразования по отношению к серверным столбцам text и ntext поддерживается.

Типы двоичных строк

Типами двоичных строк JDBC являются BINARY, VARBINARY и LONGVARBINARY.

Тип Описание

Фиксированная длина

Тип SQL Server binary непосредственно соответствует типу BINARY JDBC. Это тип фиксированной длины с заполнением, предоставляемым сервером в случае, если для столбца установлено значение SET ANSI_PADDING ON. Если на сервере заполнение для столбцов символьных типов не предусмотрено, то заполнение добавляет драйвер JDBC.

Тип SQL Server timestamp представляет собой тип BINARY JDBC с постоянной длиной 8 байт.

Переменная длина

Тип SQL Server varbinary сопоставляется с типом JDBC VARBINARY.

Тип udt в SQL Server соответствует JDBC как тип VARBINARY.

Long

Тип SQL Server image сопоставляется с типом JDBC LONGVARBINARY. Этот тип является устаревшим, начиная с версии SQL Server 2005, и вместо него следует использовать тип больших значений varbinary(max).

Точные числовые типы

Точные числовые типы JDBC непосредственно соответствуют аналогичным типами SQL Server.

Тип Описание

BIT

Тип BIT JDBC представляет единственный бит, который может иметь значение 0 или 1. Сопоставляется с типом SQL Server bit.

TINYINT

Тип TINYINT JDBC представляет единственный байт. Сопоставляется с типом SQL Server tinyint.

SMALLINT

Тип JDBC SMALLINT представляет 16-разрядное целочисленное значение со знаком. Сопоставляется с типом SQL Server smallint.

INTEGER

Тип JDBC INTEGER представляет 32-разрядное целочисленное значение со знаком. Сопоставляется с типом SQL Server int.

BIGINT

Тип JDBC BIGINT представляет 64-разрядное целочисленное значение со знаком. Сопоставляется с типом SQL Server bigint.

NUMERIC

Тип NUMERIC JDBC представляет значение десятичного числа постоянной точности, которое хранит значения идентичной точности. Тип NUMERIC сопоставляется с типом SQL Server numeric.

DECIMAL

Тип DECIMAL JDBC представляет значение десятичного числа постоянной точности, которое хранит значения по крайней мере указанной точности. Тип DECIMAL сопоставляется с типом SQL Server decimal.

Тип DECIMAL JDBC также соответствует типам SQL Server money и smallmoney, которые являются конкретными типами десятичного числа постоянной точности, хранящимися в 8 и 4 байтах соответственно.

Приблизительные числовые типы

Приблизительными числовыми типами JDBC являются REAL, DOUBLE и FLOAT.

Тип Описание

REAL

Тип REAL JDBC имеет семь цифр точности (одинарная точность) и непосредственно соотетствует типу SQL Server real.

DOUBLE

Тип DOUBLE JDBC имеет 15 цифр точности (двойная точность) и соответствует типу SQL Server float. «Тип FLOAT JDBC» — синоним «DOUBLE». В связи с возможной путаницей между FLOAT и DOUBLE предпочтительным является DOUBLE.

Типы даты и времени

Тип TIMESTAMP JDBC соответствует типам SQL Server datetime и smalldatetime. Значения типа datetime хранятся в двух четырехбайтовых целых числах. Тип smalldatetime хранит ту же информацию (дату и время), но с меньшей точностью, в двух двухбайтовых целых числах типа small.

Примечание. Тип SQL Server timestamp — это тип двоичной строки фиксированной длины. Он не соответствует ни одному типу типов JDBC для представления времени: DATE, TIME или TIMESTAMP.

Соответствие пользовательских типов

Функция сопоставления пользовательских типов JDBC, использующая интерфейсы SQLData для усовершенствованных типов JDBC (пользовательские, типы Struct и так далее), не реализованы в драйвере JDBC.
