Системные требования для драйверов JDBC

Чтобы получить доступ к данным из базы данных SQL Server с помощью драйвера JDBC Microsoft SQL Server, на компьютере должны быть установлены следующие компоненты:

  • Microsoft SQL Server Драйвер JDBC
  • Среда выполнения Java

Требования к среде выполнения Java

В драйвере Microsoft SQL Server JDBC поддержка API спецификации Java Database Connectivity была расширена и теперь включает API JDBC 4.0. JDBC 4.0 была реализована как часть Sun Java SE Development Kit (JDK) 6.0 и среды выполнения Java (JRE) 6.0. JDBC 4.0 является множеством JDBC 3.0.

Драйвер Microsoft SQL Server JDBC поддерживает JDBC 3.0 и JDBC 4.0.

При развертывании драйвера Microsoft SQL Server JDBC в операционных системах Windows и UNIX необходимо использовать пакеты установки sqljdbc_<version>_enu.exe и sqljdbc_<version>_enu.tar.gz соответственно. Дополнительные сведения о развертывании драйвера JDBC см. в разделе Развертывание драйвера JDBC.

Драйвер JDBC во всех пакетах установки содержит 2 библиотеки классов JAR: sqljdbc.jar и sqljdbc4.jar, что позволяет обеспечить поддержку обратной совместимости и возможных вариантов обновления.

JAR Описание

sqljdbc.jar

Библиотека классов sqljdbc.jar обеспечивает поддержку JDBC 3.0.

Для класса sqljdbc.jar необходима среда выполнения Java (JRE) версии 5.0. Использование sqljdbc.jar в JRE 6.0 приведет к возникновению исключения при соединении с базой данных.

Примечание. Драйвер JDBC не поддерживает JRE 1.4. При использовании драйвера JDBC необходимо обновить JRE 1.4 до версии 5.0 или более поздней. В некоторых случаях может потребоваться перекомпиляция приложения, если оно не совместимо с JDK 5.0 или более поздней версией. Дополнительные сведения см. в документации на веб-сайте Sun Microsystems.

sqljdbc4.jar

Библиотека классов sqljdbc4.jar обеспечивает поддержку JDBC 4.0. Она включает все функции sqljdbc.jar, а также новые методы JDBC 4.0.

Для класса sqljdbc4.jar необходима среда выполнения Java (JRE) версии 6.0 или более поздняя версия. Использование sqljdbc4.jar в JRE 1.4 или 5.0 приведет к возникновению исключения.

Примечание. Если необходимо использование приложения в JRE 6.0, следует использовать sqljdbc4.jar, даже если в приложении не используются функции JDBC 4.0.

Обратите внимание, что драйвер JDBC разработан и поддерживается при использовании со всеми основными аналогами виртуальных машин Java от Sun, но его проверка выполняется в Sun JRE 5.0 или более поздней версии.

Требования к SQL Server

Драйвер JDBC был разработан специально для использования функций, появившихся в SQL Server 2005, но он также обеспечивает обратную совместимость с SQL Server 2000, включая 64-разрядную версию. Версия 3.0 драйвера JDBC поддерживает новые типы даты и времени, крупные, определяемые пользователем типы и разреженные столбцы в SQL Server 2008.

Требования к операционной системе

Драйвер JDBC разработан для использования с любой операционной системой, поддерживающей использование виртуальных машин Java (JVM). Только следующие операционные системы были официально протестированы: Sun Solaris, SUSE Linux, и Windows XP и более поздник версии.

Поддерживаемые языки

Драйвер JDBC поддерживает все параметры сортировки столбцов SQL Server. Дополнительные сведения о параметрах сортировки, поддерживаемых драйвером JDBC, см. в разделе Функции поддержки различных языков драйвера JDBC.

Дополнительные сведения о параметрах сортировки см. в разделе "Использование параметров сортировки" электронной документации SQL Server.
