Работа с большими объемами данных

Начиная с версии 1.2, драйвер Microsoft SQL Server 2005 JDBC обеспечивает поддержку адаптивной буферизации, которая позволяет получать любые данные большого объема, не расходуя ресурсы на серверные курсоры. Драйвер Microsoft SQL Server JDBC с адаптивной буферизацией получает результаты выполнения инструкции с SQL Server по мере запросов приложения, а не все сразу. Драйвер также удаляет результаты, когда приложение теряет к ним доступ.

В версии 1.2 драйвера Microsoft SQL Server 2005 JDBC по умолчанию использовался режим буферизации full. Если приложение не устанавливало свойство responseBuffering в значение adaptive в свойствах соединения или с помощью метода setResponseBuffering объекта SQLServerStatement, то драйвер поддерживал одновременное считывание всех результатов с сервера. Чтобы включить режим адаптивной буферизации, приложение должно было явно устанавливать свойство соединения responseBuffering в значение adaptive.

Начиная с версии 2.0 драйвера Microsoft SQL Server JDBC, режим буферизации по умолчанию имеет значение adaptive, и драйвер JDBC выполняет буферизацию для минимально необходимого объема данных. Дополнительные сведения об использовании адаптивной буферизации см. в разделе Использование адаптивной буферизации.
