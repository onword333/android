Развертывание драйвера JDBC

При развертывании приложения, которое зависит от драйвера JDBC Microsoft SQL Server, необходимо осуществить вторичное распространение драйвера JDBC совместно с приложением. В отличие от компонентов доступа к данным Windows DAC, которые являются компонентами операционной системы Windows, драйвер JDBC считается компонентом SQL Server.

Примечание. Хотя можно свободно выполнить распределение драйвера JDBC в соответствии с требованиями приложения, необходимо сначала зарегистрироваться. Дополнительные сведения о регистрации для повторного распределения драйвера JDBC см. в разделе Регистрация для получения прав для повторного распределения драйвера Microsoft SQL Server JDBC.

Для развертывания драйвера JDBC в приложении используются два подхода. При использовании одного файлы драйвера JDBC включаются как часть собственного настраиваемого пакета установки. Во втором подходе используется пакет установки JDBC, предоставляемый Microsoft, который можно загрузить с Центра разработки драйвера JDBC для Microsoft SQL Server.

В следующих разделах описывается использование пакета установки в операционных системах Windows и UNIX.

Примечание. Дополнительные общие сведения о развертывании приложений Java см. в разделе Обзор развертывания Java на веб-сайте компании Sun Microsystems.

Развертывание драйвера JDBC в системах Windows

При развертывании драйвера драйвера JDBC в операционных системах Windows необходимо использовать исполняемый файл в формате zip пакета установки, который обычно называется sqljdbc_<version>_<language>.exe.

Для автоматического запуска исполняемого файла в формате zip необходимо использовать параметр командной строки /auto в командной строке или в пакетном файле, как описано ниже:

sqljdbc_<version>_<language>.exe /auto

Примечание. При использовании параметра /auto установка не будет полностью автоматической, поскольку на экране пользователя все же будет отображаться диалоговое окно WinZip. Однако оно не требует выполнения каких-либо действий пользователя и закрывается после выполнения операции распаковки.

Развертывание драйвера в системах UNIX

При развертывании драйвера драйвера JDBC в операционных системах UNIX необходимо использовать исполняемый файл в формате gzip пакета установки, который обычно называется sqljdbc_<version>_<language>.tar.gz.

До установки драйвера JDBC необходимо убедиться в том, что в системе пользователя установлены программы gzip и tar, и что папки, содержащие исполняемые файлы обеих программ, добавлены к переменной среды PATH.

Чтобы распаковать сжатый TAR-файл, перейдите в каталог, в который нужно распаковать драйвер, и введите следующую команду:

gzip -d sqljdbc_<version>_<language>.tar.gz

Чтобы распаковать TAR-файл, переместите его в каталог, в который нужно установить драйвер, и введите следующую команду:

tar –xf sqljdbc_<version>_<language>.tar
